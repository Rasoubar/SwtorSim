"restrictions": {-> Only include aplicable ones pls.
  "target_hp_below_pct": 0.30, -> if target is not below this hp, can't cast
  "bypass_if_buff_active": ["Bypass_Proc_Name"], -> this is only for target_hp_below_pct. bypasses that restriction if any of the buff in the list is present on caster.
  "target_has_debuff": ["Target_Debuff_Name"], -> can only cast if any of the buff in the list is present on caster.
  "caster_had_buff": ["Caster_Buff_Name"] -> can only cast if any of the debuff in the list is present on target.

"actions":Actions can be diferent types. Every type will have a diferent format.
	  The current possible actions are "damage", "dot", "buff", "debuff", "resource_gain" and "cooldown_mod"
Will now go over the formating for each kind of action.

{
"action_type": "damage" -> name of the action type. Required.
"attack_type": int -> attack type. Required. #1 = melee, 2 = ranged, 3 = force, 4 = tech
"damage_type": int -> damage type. Required if attack_type is 3 or 4. #1 = kinetic, 2 = energy, 3 = elemental, 4 = internal
"amp": float -> Required. Check jedipedia for value. 
"coeff": float -> Required. Check jedipedia for value. 
"shp_min": float -> Required. Check jedipedia for value.
"shp_max": float -> Required. Check jedipedia for value.
"delay": float -> How long after the ability is cast the action happens. Will default to 0 if not in. Check Jedipedia for number, to play safe check parses. If its stuff like a single hit when ability is cast, game can be deceiving. Contact me to know more.
"impact_delay": floar ->  How long the ability takes to land after the damage is calculated. Example is a 1st hit on a normal attack. My observation (with many tests) is that the action will happen, do procs and damage calculation but only land 0.2 seconds later. Contact me to know more.
"tags" -> array -> tags that the action has. Check jedipedia to get them (not all will be useful).
"conditions"} -> Not required. Very similar to "restrictions" for Abilities. Example below:

	"conditions": { 
	  "has_dot": true, -> the condition needs a dot (any dot) to be resolved?
 	 "exact_dot_amount": true, -> only needed if has_dot is present. Tells if the the condiction needs a certain amount of dots.
 	 "required_count": 2, -> only needed if exact_dot_amount present. Tells the number of dots
  	"has_debuff": "Target_Debuff_Name", -> If action needs certain debuff on target to resolve.
 	 "has_buff": "Caster_Buff_Name" -> If action needs certain buff on caster to resolve.
	} NOTE: there are more conditions, will be added over time by interested parties, if there are interested parties. Contact me to add.

Procs

{
  "PROC NAME": {
  "proc_name": "lame_proc_name", -> name of proc, required.
  "trigger": string, -> is it on hit or on crit? if on hit, put "hit", if on crit put "crit"
  "required_tag": "elemental", -> if the hit needs to have a tag, put it here. Not Required.
  "chance": float, -> if it's got random chance, put it here. use 0.X where X is percentage. Not Required, will default to 1.0
  "icd": float, -> cooldown of the proc. will default to 0. Not required
  "affected_by_cdr": bool, - > if affected by internal cooldown affected by cooldown reduction put TRUE, if not put FALSE, defaults to FALSE. Not required.
  "action": {} -> what it actualy does. format is the same as action (single) from abilities. 
  }
}
